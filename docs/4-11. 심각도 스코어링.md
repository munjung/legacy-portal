# 4-11. 심각도 스코어링

[4-10. 도출 스멜 - 심각도 표](./4-10.%20도출%20스멜%20-%20심각도%20표.md)에서 나온 스멜 목록을 **영향도 × 수정난이도 매트릭스**에 배치해 우선순위를 도출한다. 이전 버전(범위·위험도·선행조건 3축 합산 점수)은 "지금 얼마나 심각한가"만 봤다면, 이 버전은 "심각하더라도 고치기 쉬운가"까지 함께 봐서 **실제로 뭘 먼저 손댈지**를 정하는 데 초점을 둔다. 새로운 탐지는 하지 않고 4-4~4-10의 근거를 그대로 매트릭스 좌표로 옮긴 것이다.

## 두 축의 정의

| 축 | 상 | 중 | 하 |
|---|---|---|---|
| **영향도** (방치 시 위험·범위) | 여러 도메인/서비스에 퍼져 있고 방치 시 버그·안전망 부재로 이어짐 | 한 클래스 안에 국한되거나, 넓게 퍼졌어도 당장 버그로 이어지진 않음 | 가독성 문제에 그치고 동작에는 영향 없음 |
| **수정난이도** (CLAUDE.md 불변 규칙 하에서 고치는 데 드는 위험·노력) | 로직이 얽혀 있어 손대면 다른 동작을 건드릴 위험이 큼(특성화 테스트 없이는 특히 위험) | 기계적으로 처리 가능하지만 빠짐없이 챙겨야 할 지점이 여럿 | 단순 치환·이동 수준으로 위험이 거의 없음 |

## 영향도 × 수정난이도 매트릭스

|  | 난이도 하 | 난이도 중 | 난이도 상 |
|---|---|---|---|
| **영향도 상** | *(해당 없음)* | **No Tests①**, Duplicated Code | God Class, Long Method |
| **영향도 중** | Tight Coupling, Feature Envy | Magic Number / Primitive Obsession | *(해당 없음)* |
| **영향도 하** | Poor Naming, Comment Smell, Long Parameter List | *(해당 없음)* | *(해당 없음)* |

① No Tests는 매트릭스 상 Duplicated Code와 같은 칸이지만, 이 항목만은 **다른 모든 항목을 안전하게 고치기 위한 선결 조건**이라는 예외적 위치에 있다 — 매트릭스 좌표와 별개로 항상 최우선이다([4-10](./4-10.%20도출%20스멜%20-%20심각도%20표.md) 참고).

## 매트릭스에서 도출한 우선순위

| 순위 | 영향도 × 난이도 | 스멜 | 착수 방식 |
|---|---|---|---|
| 1 | 상 × 중 (단, 선결조건이라 최우선) | No Tests | 다른 어떤 것보다 먼저 — 특성화 테스트부터 확보 |
| 2 | 상 × 중 | Duplicated Code | 감사로그·메일·권한판정 등 공통 모듈 추출은 기계적이라 테스트 확보 직후 바로 착수 가능 |
| 3 | 상 × 상 | God Class, Long Method | 영향은 크지만 얽혀 있어 신중한 계획 필요 — 테스트가 있는 상태에서만 착수(사실상 `ApprovalService` 전체를 다루는 하나의 과제) |
| 4 | 중 × 하 | Tight Coupling, Feature Envy | 쉬우니 위 작업과 병행하거나 그 사이사이에 처리 가능 |
| 5 | 중 × 중 | Magic Number / Primitive Obsession | 범위는 넓지만 `@Convert`+`AttributeConverter`로 기계적으로 흡수 — 여유 될 때 진행 |
| 6 | 하 × 하 | Poor Naming, Comment Smell, Long Parameter List | 위 작업들을 진행하며 손대는 코드에 한해 자연스럽게 함께 정리 |

## 매트릭스가 알려주는 것

- **왼쪽 위(고영향·저난이도) 칸이 비어 있다** — 이 코드베이스에는 "심각한데 고치기도 쉬운" 손쉬운 승부수가 없다. 가장 심각한 문제(No Tests, God Class, Long Method)일수록 고치기도 까다롭다는 뜻이라, 서두르기보다 순서(테스트 → 중복 제거 → God Class/Long Method 분해)를 지키는 편이 안전하다.
- **오른쪽(고난이도) 칸에는 영향도 상만 있다** — 중간 이하 영향도 항목은 전부 난이도가 하~중이라, 우선순위 다툼이 크지 않다. 즉 "상" 등급 세 가지(No Tests·Duplicated Code·God Class/Long Method)만 순서를 잘 지키면 나머지는 순서 영향이 적다.
- **Duplicated Code가 God Class·Long Method보다 먼저 오는 이유**: 영향도는 같은 "상"이지만 난이도가 한 단계 낮다(공통 모듈 추출은 상태 전이 로직 자체를 안 건드림) — 매트릭스가 정성적 등급표(4-10)만으로는 안 보이던 착수 순서를 드러내는 지점이다.

## 참고

매트릭스 좌표·순위는 참고용이며, 실제 착수 순서·리팩토링 기법 선택은 CLAUDE.md 불변 규칙(API·시그니처·DB 저장값·관찰 가능한 동작 보존)과 함께 별도 단계에서 확정한다.
